# V008 在 Ubuntu 下的运行指南

本指南针对 **Ubuntu/Linux 环境**，说明如何在本机运行 Grain Agent **V008**，并进行基本测试。

---

## 📋 前置准备

### 1. 检查 Python 环境

确保已安装 Python 3.8+（建议使用系统自带或 Anaconda）：

```bash
python --version
# 或
python3 --version
```

后续命令中，如果你的系统里 `python` 指向 Python2，可以把下面所有的 `python`/`pip` 换成 `python3`/`pip3`。

### 2. 安装依赖

在项目根目录下执行：

```bash
cd V008
pip install -r requirements.txt
```

### 3. 配置环境变量

V008 需要 `DASHSCOPE_API_KEY`（通义千问 API Key）。

**方式一：终端中临时设置（当前终端有效）**

```bash
export DASHSCOPE_API_KEY="your-api-key-here"
export LLM_MODEL="qwen-max"
export DEBUG="false"
```

**方式二：使用 `.env` 文件（推荐）**

在 `V008` 目录下创建 `.env`：

```text
DASHSCOPE_API_KEY=your-api-key-here
LLM_MODEL=qwen-max
DEBUG=false
```

### 4. 验证环境变量是否生效

```bash
cd V008
python - << 'PY'
import os
from dotenv import load_dotenv
load_dotenv()
print("DASHSCOPE_API_KEY:", "已设置" if os.getenv("DASHSCOPE_API_KEY") else "未设置")
print("LLM_MODEL:", os.getenv("LLM_MODEL", "未设置"))
PY
```

## 🚀 启动服务（Ubuntu）

### 方式一：使用 `start_server.sh`（⭐ 推荐）

`V008/start_server.sh` 会自动清理端口占用和残留 uvicorn 进程，并写日志/PID 文件。

```bash
cd V008
chmod +x start_server.sh          # 第一次使用时
./start_server.sh                 # 等价于 ./start_server.sh 8000 0.0.0.0 false
# 或显式指定：
# ./start_server.sh 8000 0.0.0.0 true
```

脚本会：

- 自动设置 `PYTHONIOENCODING=utf-8`
- 将日志写入 `V008/server.log`
- 将 PID 写入 `V008/.server.pid`

服务启动后：

- API 文档（`DEBUG=true`）：`http://127.0.0.1:8000/docs`
- 健康检查：`http://127.0.0.1:8000/`
- 统一接口：`POST http://127.0.0.1:8000/api/v1/agent/chat`

### 方式二：直接使用 uvicorn

```bash
cd V008
export PYTHONIOENCODING="utf-8"
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
# 或
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

## 🛑 停止服务（Ubuntu）

### 方式一：使用 `stop_server.sh`（⭐ 推荐）

```bash
cd V008
chmod +x stop_server.sh    # 第一次使用时
./stop_server.sh
```

脚本会根据 `.server.pid` 优雅发送 SIGTERM，必要时强制 kill，并清理 PID 文件/残留进程。

### 方式二：前台运行时 `Ctrl + C`

如果是前台跑的 `uvicorn ... --reload`，直接在终端按 `Ctrl + C` 即可。

```bash
  -H "Content-Type: application/json" \
  -d '{"query": "生成1号仓的日报"}'
```

### 方法三：Python requests

```python
import requests

url = "http://127.0.0.1:8000/api/v1/agent/chat"
headers = {"Content-Type": "application/json"}

queries = [
    "1号仓的粮温情况如何？",
    "巡检一下所有粮仓",
    "给我生成1号仓的温度曲线图",
    "生成1号仓的日报",
]

for q in queries:
    print("\n" + "=" * 60)
    print("查询:", q)
    print("=" * 60)
    resp = requests.post(url, json={"query": q}, headers=headers, timeout=60)
    print("状态码:", resp.status_code)
    print(resp.text)
```

## ⚠️ 常见问题（Ubuntu）

- **缺少 `python-docx`**

  ```bash
  pip install "python-docx>=1.1.0"
  ```

- **编码报错**

  ```bash
  export PYTHONIOENCODING="utf-8"
  ```

- **`DASHSCOPE_API_KEY` 未设置**  
  按前文方式设置环境变量或 `.env` 即可。

- **端口被占用 / uvicorn 残留**

  ```bash
  cd V008
  ./stop_server.sh          # 优先用脚本
  lsof -i:8000              # 手动查看
  pgrep -a uvicorn          # 查找 uvicorn
  kill <PID> || kill -9 <PID>
  ./start_server.sh 8000 0.0.0.0 true
  ```
