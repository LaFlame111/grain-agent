# RAGFlow 操作指南（从零上手）

> 本指南面向新手用户，涵盖 RAGFlow 从启动到使用的完整流程。

---

## 第一步：启动服务

### 1.1 启动 Docker Desktop

- 双击桌面上的 **Docker Desktop** 图标
- 等待左下角状态变为绿色 **"Engine running"**，表示 Docker 已就绪

### 1.2 启动 RAGFlow

打开终端（PowerShell 或 Git Bash），执行：

```bash
cd C:\Users\ASUS\Desktop\ragflow\docker
docker compose up -d
```

等待几分钟，直到 5 个容器全部启动：

| 容器            | 作用              |
| --------------- | ----------------- |
| ragflow         | 主服务（API + Web） |
| mysql           | 数据库            |
| elasticsearch   | 搜索引擎          |
| minio           | 文件存储          |
| redis           | 缓存              |

### 1.3 验证服务是否正常

```bash
docker compose ps
```

确认 5 个容器状态都是 **running**。

---

## 第二步：登录 Web 界面

1. 打开浏览器，访问 **http://localhost**
2. 首次使用需要 **注册账号**（自己设置用户名和密码即可，数据存在本地）
3. 注册后直接登录

---

## 第三步：创建知识库

1. 点击左侧菜单的 **"知识库"**
2. 点击右上角 **"创建知识库"** 按钮
3. 填写信息：
   - **知识库名称**：起一个有意义的名字（如 `grain_knowledge`）
   - **嵌入模型**：选择已配置好的模型（如果没有，需要先在"模型管理"中配置）
   - **分块方式**：根据文档类型选择（不确定就选 **General**）
4. 点击 **"确定"** 创建

---

## 第四步：上传文档

1. 进入刚创建的知识库
2. 点击 **"上传文件"** 按钮
3. 选择文件（支持 PDF、Word、Markdown、TXT、Excel 等）
4. **重要提醒**：
   - 建议 **一次只上传一个文件**，特别是 PDF，解析很吃资源
   - 上传后文件状态会显示为 **"待解析"**

---

## 第五步：解析文档

1. 上传完成后，勾选文件前面的复选框
2. 点击上方的 **"解析"** 按钮（或者点击播放 ▶ 图标）
3. 等待解析完成，状态会变为 **"已完成"**
4. 点击文件名可以查看 **分块结果**：
   - 每个分块的内容预览
   - 分块数量
   - 如果分块效果不满意，可以 **删除后换一种分块方式重新上传解析**

### 如何判断分块质量？

- 每个分块应该包含 **完整的语义信息**（不要把一句话切成两半）
- 分块不要太长（超过 500 字检索精度下降）也不要太短（几个字没有意义）
- 表格数据要保持行列对应关系

---

## 第六步：配置模型（首次需要）

如果还没配置过 AI 模型：

1. 点击右上角 **头像** → **"模型管理"** 或 **"系统设置"**
2. 添加模型供应商（如 OpenAI、通义千问、本地 Ollama 等）
3. 需要配置两种模型：
   - **嵌入模型（Embedding）**：用于把文本转成向量，知识库检索用
   - **对话模型（Chat/LLM）**：用于生成回答

| 模型类型   | 作用               | 示例                                       |
| ---------- | ------------------ | ------------------------------------------ |
| 嵌入模型   | 文本→向量，做检索  | `text-embedding-ada-002`、`bge-large-zh`    |
| 对话模型   | 理解问题，生成回答 | `gpt-3.5-turbo`、`qwen-plus`、`deepseek`    |

---

## 第七步：创建聊天助手

1. 点击左侧菜单的 **"聊天"**
2. 点击 **"创建助手"**
3. 配置：
   - **助手名称**：如 `粮储知识助手`
   - **选择对话模型**：选择第六步配置好的 LLM
   - **关联知识库**：勾选你创建的知识库（如 `grain_knowledge`）
   - **系统提示词（可选）**：告诉 AI 它的角色，例如：
     ```
     你是一个粮油储藏领域的专业顾问，请基于知识库内容准确回答用户问题。
     如果知识库中没有相关信息，请如实告知。
     ```
4. 其他参数（可先用默认值）：
   - **相似度阈值**：控制检索匹配的严格程度（越高越严格）
   - **Top K**：返回最相关的分块数量（一般 3-5）
5. 点击 **确定** 创建

---

## 第八步：开始对话测试

1. 点击刚创建的助手，进入对话界面
2. 输入问题，例如：
   - `"粮食储存的安全温度是多少？"`
   - `"机械通风的操作规程是什么？"`
3. 系统会自动：
   - 从知识库中检索相关分块
   - 将分块内容交给 LLM 生成回答
4. 回答下方通常会显示 **引用来源**，可以点击查看原文

---

## 第九步：停止服务

用完后关闭 RAGFlow：

```bash
cd C:\Users\ASUS\Desktop\ragflow\docker
docker compose down
```

> **注意**：`down` 只是停止容器，**数据不会丢失**（存在 Docker volume 中）。下次 `up -d` 数据还在。

---

## 常见问题排查

| 问题                        | 解决方案                                                       |
| --------------------------- | -------------------------------------------------------------- |
| 浏览器打不开 localhost       | 检查 Docker 容器是否全部 running，等几分钟再试                  |
| PDF 解析卡住不动             | PDF 解析很吃资源，耐心等待；如果 Docker 崩溃，重启后重新解析    |
| 检索结果不相关               | 换一种分块方式重新解析，或调整相似度阈值和 Top K                |
| 回答说"未找到相关信息"       | 检查知识库是否关联到助手、文档是否解析成功                      |
| Docker Desktop 启动慢/崩溃  | 确保电脑至少 **16GB 内存**，关闭其他占资源的程序                |

## 操作流程总结图

```
启动 Docker Desktop
       ↓
docker compose up -d（启动5个容器）
       ↓
浏览器打开 http://localhost
       ↓
创建知识库 → 选择分块方式
       ↓
上传文档 → 解析文档 → 检查分块质量
       ↓
配置模型（嵌入模型 + 对话模型）
       ↓
创建聊天助手 → 关联知识库
       ↓
开始对话测试
       ↓
docker compose down（用完关闭）
```
