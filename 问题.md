# V008 版本问题与改进跟踪

本文档用于记录 V008 版本的问题与改进事项。以下内容包含一部分 V007 历史问题的备查记录，**状态未验证**，仅供追踪参考。

## 历史问题记录（V007 备查）

根据 `time-range-data-visualization` 技能和项目规范，对 V007 版本进行了反复核验，发现以下问题：

## 1. 核心业务逻辑问题

### 1.1 可视化工具（T7）未实现 7 个数据点的限制
- **描述**：`GrainTools.visualization` 方法（用于生成平均粮温趋势图）未实现 7 个数据点的限制逻辑。
- **位置**：`V007/app/services/tools.py` 中的 `visualization` 方法（第 348-493 行）。
- **预期行为**：根据技能规范，如果数据点超过 7 个，应根据是否指定了时间范围来决定取前 7 个（最早）或后 7 个（最晚）数据。
- **对比**：新增的 `generate_three_temp_chart` 和 `generate_two_humidity_chart` 已正确实现此逻辑。

## 2. 测试脚本问题

### 2.1 `verify_chart_dates.py` 运行失败
- **描述**：运行 `python V007/verify_chart_dates.py` 时报错 `错误: 没有找到仓号`。
- **原因**：脚本直接实例化了 `DataLoader` 但未调用 `load()` 方法，导致索引为空。
- **位置**：`V007/verify_chart_dates.py` 第 20-23 行。
- **建议**：应使用 `get_data_loader()` 获取单例实例，或者手动调用 `data_loader.load()`。

## 3. 文档与一致性问题

### 3.1 三温图和两湿图未作为独立工具暴露给 LLM
- **描述**：虽然 `tools.py` 中实现了 `generate_three_temp_chart` 和 `generate_two_humidity_chart`，且在 `report` 工具中被调用，但它们并未在 `tool_definitions.py` 中被定义为独立的工具。
- **影响**：用户无法直接要求 Agent “生成三温图”，只能通过“生成报告”来间接获取。
- **建议**：如果这两个图表是核心能力，建议在 `tool_definitions.py` 中增加对应的工具定义。

## 4. 代码健壮性问题

### 4.1 数据加载器的单例初始化警告
- **描述**：如果数据加载失败，`DataLoader` 仅打印警告并继续，这可能导致后续查询返回空结果。虽然有 Mock 回退机制，但日志中应更清晰地记录回退发生的原因。
- **位置**：`V007/app/services/data_loader.py` 第 231-232 行。
