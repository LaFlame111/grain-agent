# V006 中间件日志查看指南

## 📋 中间件2的日志配置

中间件2（自定义HTTP中间件）的日志配置在 `app/main.py` 中：

```python
# 第10-13行：日志配置
logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s - %(name)s - %(levelname)s - %(message)s",
)
```

**关键点：**
- 没有指定 `filename` 参数
- 日志输出到**标准输出（控制台）**
- 不会自动写入文件

## 🔍 如何查看日志

### 方法 1: 直接查看控制台（最简单）

**启动服务时，日志会实时显示在终端中：**

```powershell
# PowerShell
cd V006
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

**日志会实时显示：**
- ✅ 请求开始日志
- ✅ 请求完成日志
- ✅ 请求失败日志（如果有）

---

### 方法 2: 重定向输出到文件（推荐）

**将控制台输出保存到文件：**

#### Windows PowerShell:
```powershell
# 启动服务并保存日志到文件
cd V006
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload *> app.log

# 或者只保存标准输出（不包括错误）
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload > app.log 2>&1
```

#### Windows CMD:
```cmd
cd V006
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload > app.log 2>&1
```

#### Linux/Mac:
```bash
cd V006
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload > app.log 2>&1
```

**查看日志文件：**
```powershell
# 查看完整日志
Get-Content app.log

# 查看最后50行
Get-Content app.log -Tail 50

# 实时查看（类似 tail -f）
Get-Content app.log -Wait -Tail 20
```

---

### 方法 3: 使用后台运行并保存日志

**Windows PowerShell:**
```powershell
# 后台运行并保存日志
cd V006
Start-Process python -ArgumentList "-m", "uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000" -RedirectStandardOutput "app.log" -RedirectStandardError "app_error.log" -NoNewWindow
```

**Linux/Mac:**
```bash
# 使用 nohup 后台运行
cd V006
nohup uvicorn app.main:app --host 0.0.0.0 --port 8000 > app.log 2>&1 &

# 查看日志
tail -f app.log
```

## 🔧 修改日志配置（输出到文件）

如果您希望日志自动保存到文件，可以修改 `app/main.py`：

### 方案 1: 同时输出到控制台和文件

```python
import logging
from logging.handlers import RotatingFileHandler

# 配置日志格式
log_format = "%(asctime)s - %(name)s - %(levelname)s - %(message)s"

# 配置根日志记录器
logging.basicConfig(
    level=logging.INFO,
    format=log_format,
    handlers=[
        logging.StreamHandler(),  # 输出到控制台
        RotatingFileHandler(
            "logs/app.log",
            maxBytes=10*1024*1024,  # 10MB
            backupCount=5,
            encoding="utf-8"
        )  # 输出到文件
    ]
)

logger = logging.getLogger(__name__)
```

### 方案 2: 只输出到文件

```python
import logging
from logging.handlers import RotatingFileHandler

# 确保日志目录存在
import os
os.makedirs("logs", exist_ok=True)

# 配置日志格式
log_format = "%(asctime)s - %(name)s - %(levelname)s - %(message)s"

# 配置根日志记录器
logging.basicConfig(
    level=logging.INFO,
    format=log_format,
    handlers=[
        RotatingFileHandler(
            "logs/app.log",
            maxBytes=10*1024*1024,  # 10MB
            backupCount=5,
            encoding="utf-8"
        )
    ]
)

logger = logging.getLogger(__name__)
```

**日志文件位置：**
- `V006/logs/app.log` - 当前日志
- `V006/logs/app.log.1` - 备份日志（当文件超过10MB时）

## 📝 日志格式说明

当前日志格式：
```
%(asctime)s - %(name)s - %(levelname)s - %(message)s
```

**格式说明：**

| 字段 | 说明 | 示例 |
|------|------|------|
| `%(asctime)s` | 时间戳 | `2025-12-18 19:00:15,456` |
| `%(name)s` | 日志记录器名称 | `__main__` |
| `%(levelname)s` | 日志级别 | `INFO`, `ERROR`, `WARNING` |
| `%(message)s` | 日志消息 | `Request started: POST /api/v1/agent/chat [ID: ...]` |

## ⚠️ 注意事项

1. **日志不会自动保存**
   - 当前配置只输出到控制台
   - 需要手动重定向或修改配置才能保存到文件

2. **日志文件可能很大**
   - 如果使用文件输出，建议使用 `RotatingFileHandler` 自动轮转
   - 避免日志文件无限增长

3. **日志级别**
   - 当前设置为 `INFO` 级别
   - 只会记录 INFO、WARNING、ERROR 级别的日志
   - DEBUG 级别的日志不会显示

4. **编码问题**
   - Windows 可能需要设置编码：`$env:PYTHONIOENCODING="utf-8"`
   - 日志文件建议使用 UTF-8 编码
