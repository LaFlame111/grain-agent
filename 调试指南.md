# V006 断点调试指南

## 🔍 为什么断点没有触发？

### 常见原因分析

1. **没有通过调试器启动应用**
   - ❌ 直接在终端运行 `uvicorn app.main:app` - 这样启动的应用没有调试器附加，断点不会触发
   - ✅ 应该通过 VS Code 的调试功能启动 - 这样调试器会附加到进程，断点才能正常工作

2. **调试器配置不正确**
   - ❌ `justMyCode: true`（默认值）- 只调试自己的代码，不进入库函数
   - ✅ `justMyCode: false` - 允许进入所有代码，包括库函数和方法

3. **工作目录或路径配置错误**
   - ❌ Python 找不到模块
   - ✅ 需要正确设置 `cwd` 和 `PYTHONPATH`

4. **断点位置不当**
   - ❌ 断点设置在字符串字面量内部
   - ✅ 断点应该设置在可执行代码行

## 🛠️ Step-by-Step 调试方案

## 📖 如何通过 VS Code 进行调试（详细步骤）

### 第一步：打开调试面板

**方法 1：使用快捷键**
- 按 `F5` 键（最快速）
- 或按 `Ctrl+Shift+D`（打开调试视图）

**方法 2：使用菜单**
- 点击左侧活动栏的"运行和调试"图标（🐛 虫子图标）
- 或菜单：`查看` → `运行和调试`

**方法 3：使用命令面板**
- 按 `Ctrl+Shift+P` 打开命令面板
- 输入 "Debug: Start Debugging" 并回车

### 第二步：选择调试配置

1. **查看调试配置下拉菜单**
   - 在调试面板顶部，你会看到一个下拉菜单（默认可能显示 "No Configurations"）
   - 点击下拉菜单，应该能看到以下选项：
     - `Python: FastAPI (V006)` - 带自动重载的配置
     - `Python: FastAPI (V006) - No Reload` - **推荐**，不带自动重载（断点更稳定）
     - `Python: Current File` - 调试当前打开的 Python 文件

2. **选择配置**
   - **推荐选择**：`Python: FastAPI (V006) - No Reload`
   - 原因：`--reload` 选项会在代码更改时重启服务，可能导致断点失效

3. **如果看不到配置选项**
   - 检查 `.vscode/launch.json` 文件是否存在
   - 如果不存在，VS Code 会提示创建，选择 "Python" → "FastAPI"
   - 或者手动创建 `.vscode/launch.json` 文件（已为你创建好了）

### 第三步：设置断点

1. **打开要调试的文件**
   - 打开 `V006/app/services/tools.py`
   - 滚动到第 508 行（或你想调试的任何位置）

2. **设置断点**
   - **方法 1**：点击行号左侧的空白区域
   - **方法 2**：将光标放在该行，按 `F9` 键
   - **方法 3**：右键点击行号，选择 "切换断点"

3. **验证断点已设置**
   - 行号左侧应该出现一个**红色圆点** 🔴
   - 如果圆点是灰色的，说明断点未绑定（可能是路径问题）

4. **推荐断点位置**
   - ✅ **第 508 行**：`query_text = f"请对{silo_id}号仓的粮情进行综合分析..."`
   - ✅ **第 514 行**：`reasoning_result = self.llm_reasoning(`
   - ✅ **第 301 行**：`def llm_reasoning(self, query: str, context: Dict[str, Any]):`（函数入口）

### 第四步：启动调试会话

1. **启动调试**
   - 确保已选择正确的调试配置（`Python: FastAPI (V006) - No Reload`）
   - 按 `F5` 键启动调试
   - 或点击调试面板顶部的绿色播放按钮 ▶️

2. **观察调试输出**
   - VS Code 底部会打开"调试控制台"（Debug Console）
   - 终端（Terminal）会显示 uvicorn 的启动日志
   - 等待看到类似以下输出：
     ```
     INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
     INFO:     Started reloader process [xxxxx]
     INFO:     Started server process [xxxxx]
     INFO:     Waiting for application startup.
     INFO:     Application startup complete.
     ```

3. **验证调试器状态**
   - 查看 VS Code 底部状态栏（Status Bar）
   - 应该显示：`🐛 Python Debugger: Active` 或类似信息
   - 如果显示 "Waiting for debugger attach"，等待几秒钟

### 第五步：触发断点

1. **准备测试请求**
   - 打开新的终端窗口（不要关闭调试终端）
   - 或使用已有的终端运行测试脚本

2. **发送请求触发断点**
   
   **方法 1：使用 test_requests.py**
   ```bash
   # 在新终端中运行
   cd V006
   python test_requests.py
   ```
   
   **方法 2：使用 curl**
   ```bash
   curl -X POST http://localhost:8000/api/v1/agent/chat \
     -H "Content-Type: application/json" \
     -d "{\"query\": \"请生成1号仓的日报报告\"}"
   ```
   
   **方法 3：使用 Python 交互式**
   ```python
   import requests
   response = requests.post(
       "http://localhost:8000/api/v1/agent/chat",
       json={"query": "请生成1号仓的日报报告"}
   )
   print(response.json())
   ```

3. **观察断点触发**
   - 当代码执行到断点位置时，VS Code 会自动暂停执行
   - 断点所在行会**高亮显示**（通常是黄色或橙色背景）
   - 左侧会出现一个**黄色箭头** → 指向当前执行的代码行
   - 调试面板会显示：
     - **Variables（变量）**：显示当前作用域的所有变量
     - **Call Stack（调用栈）**：显示函数调用链
     - **Breakpoints（断点）**：显示所有设置的断点

### 第六步：Step-by-Step 调试

当代码在断点处暂停后，你可以使用以下快捷键进行单步调试：

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| **F10** | Step Over（单步跳过） | 执行当前行，**不进入**函数内部。如果当前行是函数调用，会直接执行完整个函数 |
| **F11** | Step Into（单步进入） | **进入**当前行的函数内部。如果当前行是 `self.llm_reasoning(...)`，按 F11 会进入 `llm_reasoning` 方法 |
| **Shift+F11** | Step Out（单步跳出） | 执行完当前函数，返回到调用处。适合快速跳出深层函数调用 |
| **F5** | Continue（继续执行） | 继续执行直到下一个断点或程序结束 |
| **Ctrl+Shift+F5** | Restart（重启调试） | 重新启动整个调试会话 |
| **Shift+F5** | Stop（停止调试） | 停止调试会话 |

**调试示例流程：**

1. **断点触发**：代码在第 508 行暂停
2. **查看变量**：在 Variables 面板查看 `silo_id` 的值
3. **执行当前行**：按 `F10`，执行 `query_text = f"..."`
4. **查看结果**：在 Variables 面板查看 `query_text` 的值
5. **进入函数**：按 `F10` 到第 514 行，然后按 `F11` 进入 `llm_reasoning` 方法
6. **单步调试函数内部**：在函数内部使用 `F10` 逐行执行
7. **跳出函数**：按 `Shift+F11` 返回到调用处
8. **继续执行**：按 `F5` 继续执行直到下一个断点

### 第七步：查看和修改变量

1. **查看变量**
   - 左侧 "Variables" 面板显示当前作用域的所有变量
   - 展开对象可以查看其属性值
   - 变量值会实时更新

2. **查看调用栈**
   - 左侧 "Call Stack" 面板显示函数调用链
   - 点击任意调用栈帧，可以查看该帧的变量和代码
   - 例如：`report` → `llm_reasoning` → `analyze`

3. **添加监视表达式**
   - 在 "Watch" 面板点击 "+" 添加表达式
   - 例如：`silo_id`, `query_text`, `reasoning_result.get("response")`
   - 监视表达式会实时计算并显示结果

4. **修改变量值（调试时）**
   - 在 Variables 面板，右键点击变量
   - 选择 "Set Value"（设置值）
   - 输入新值，可以测试不同情况下的代码行为

### 第八步：使用调试控制台

当代码在断点处暂停时，你可以在调试控制台执行 Python 代码：

1. **打开调试控制台**
   - 点击底部面板的 "Debug Console" 标签
   - 或使用快捷键 `Ctrl+Shift+Y`

2. **执行代码**
   - 输入 Python 代码并回车执行
   - 例如：
     ```python
     print(query_text)
     print(type(reasoning_result))
     len(findings)
     ```

3. **查看输出**
   - 代码执行结果会立即显示在控制台
   - 可以用于快速测试和验证

## 🐛 常见问题排查

### 问题 1: 断点显示为灰色圆圈

**原因**：断点未绑定（代码未加载或路径不匹配）

**解决**：
1. 确保通过调试器启动应用（不是直接运行）
2. 检查 `cwd` 和 `PYTHONPATH` 配置
3. 重启调试会话

### 问题 2: 断点不暂停，直接跳过

**原因**：`justMyCode: true` 且断点在库函数中

**解决**：
1. 设置 `justMyCode: false`（已在 launch.json 中配置）
2. 确保断点设置在可执行代码行（不是字符串字面量）

### 问题 3: 进入函数后无法看到变量

**原因**：函数参数或局部变量未正确加载

**解决**：
1. 使用 F10 单步执行，等待变量加载
2. 在 "Variables" 面板中展开 "Local" 作用域
3. 检查函数签名是否正确

### 问题 4: 调试器启动后立即退出

**原因**：模块导入错误或配置错误

**解决**：
1. 检查终端输出的错误信息
2. 验证 `PYTHONPATH` 和 `cwd` 配置
3. 确保所有依赖已安装

## 📝 调试技巧

### 技巧 1: 条件断点

右键点击断点 → "Edit Breakpoint" → 输入条件：
- `silo_id == "1"` - 只在处理1号仓时暂停
- `len(findings) > 5` - 只在发现数量大于5时暂停

### 技巧 2: 日志断点

右键点击断点 → "Edit Breakpoint" → "Logpoint"：
- 输入日志消息：`"silo_id: {silo_id}, query: {query_text}"`
- 不会暂停执行，但会输出日志

### 技巧 3: 临时断点

- 按 `F9` 切换当前行的断点
- 或使用 `Ctrl+F9` 禁用所有断点

### 技巧 4: 调试控制台

- 在暂停状态下，可以在 "Debug Console" 中执行 Python 代码
- 例如：`print(query_text)`, `type(reasoning_result)`
