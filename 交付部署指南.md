# 粮情分析智能体 (Grain Agent V010) — 交付部署指南

## 一、交付物清单

| 文件/目录 | 内容 | 大小 |
|-----------|------|------|
| 项目根目录 | Agent 后端源码 + 前端 + 知识库原始资料 + 测试集 | ~200 MB |
| `ragflow_docker/` | RAGFlow 部署配置（docker-compose.yml 等） | ~5 MB |
| `ragflow_export/` | Docker 镜像 + 已解析知识库数据卷（免拉取、免解析） | ~4.4 GB |

### ragflow_export 包内容

| 文件 | 大小 | 说明 |
|------|------|------|
| `images.tar` | ~4.0 GB | 5 个 Docker 镜像（RAGFlow v0.24.0 + MySQL + ES + MinIO + Redis） |
| `vol_mysql.tar.gz` | ~16 MB | MySQL 数据（用户账号、数据集配置、文档元信息） |
| `vol_es.tar.gz` | ~76 MB | Elasticsearch 数据（向量索引，4001 个 chunks） |
| `vol_minio.tar.gz` | ~339 MB | MinIO 数据（上传的原始文件副本） |
| `vol_redis.tar.gz` | ~16 KB | Redis 缓存数据 |

## 二、环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11（需启用 WSL2） |
| Python | 3.10+（脚本自动安装） |
| Docker Desktop | 29.x+（脚本自动安装） |
| 内存 | 建议 16GB+，WSL2 分配至少 12GB |
| 磁盘 | RAGFlow 容器约占 10-15GB |
| 网络 | 需访问通义千问 API（dashscope.aliyuncs.com）和 WMS 接口 |

> 不需要能访问 Docker Hub — 镜像已包含在交付包中。

## 三、一键部署（推荐）

### 步骤

```
1. git clone https://github.com/LaFlame111/grain-agent.git
2. 右键 setup.ps1 → "以管理员身份运行"
3.（若首次启用 WSL2）重启后再运行一次 setup.ps1
4. 等待自动完成（5~15 分钟）
5. 按提示输入 DASHSCOPE_API_KEY（通义千问 API Key）
6. 脚本自动获取 RAGFlow API Key 和数据集 ID 并写入 .env
7. 运行 .\start_server.ps1 启动 Agent
8. 浏览器打开 http://127.0.0.1:8000/ui
```

`setup.ps1` 会自动完成以下工作：
- 检测/启用 WSL2
- 检测/安装 Docker Desktop
- 配置 WSL2 内存（12GB）
- 检测/安装 Python 3.10+
- 导入 Docker 镜像（离线，从 `ragflow_export/`）
- 启动 RAGFlow 容器并导入知识库数据
- 安装 Python 依赖
- 配置 `.env`（交互式输入 DASHSCOPE_API_KEY + 自动获取 RAGFlow API Key）

**日志文件**：部署过程完整记录在 `setup.log`，出问题时可发此文件排查。

## 四、手动部署（备用）

如果 `setup.ps1` 遇到问题，可按以下步骤手动部署。前提：已安装 Docker Desktop 和 Python 3.10+。

### 4.1 导入 Docker 镜像

```bash
# 如果只有分卷文件，先合并
# cat ragflow_export/images.tar.part-* > ragflow_export/images.tar

# 导入镜像（约 2-5 分钟）
docker load -i ragflow_export/images.tar

# 验证
docker images | grep -E "ragflow|mysql|elasticsearch|minio|valkey"
```

### 4.2 启动 RAGFlow 创建数据卷

```bash
cd ragflow_docker
docker compose up -d
# 等待 MySQL 容器 healthy
docker compose ps
docker compose down
```

### 4.3 导入知识库数据

```powershell
# PowerShell（推荐）— 使用 RAGFlow 镜像而非 alpine
$img = "infiniflow/ragflow:v0.24.0"
$export = (Resolve-Path ".\ragflow_export").Path

docker run --rm -v "ragflow_docker_mysql_data:/data" -v "${export}:/backup:ro" $img sh -c "cd /data && tar xzf /backup/vol_mysql.tar.gz"
docker run --rm -v "ragflow_docker_esdata01:/data" -v "${export}:/backup:ro" $img sh -c "cd /data && tar xzf /backup/vol_es.tar.gz"
docker run --rm -v "ragflow_docker_minio_data:/data" -v "${export}:/backup:ro" $img sh -c "cd /data && tar xzf /backup/vol_minio.tar.gz"
docker run --rm -v "ragflow_docker_redis_data:/data" -v "${export}:/backup:ro" $img sh -c "cd /data && tar xzf /backup/vol_redis.tar.gz"
```

> 注意数据卷名称前缀 `ragflow_docker_` 必须与 Docker Compose 项目名一致。

### 4.4 启动 RAGFlow

```bash
cd ragflow_docker
docker compose up -d
# 等待所有容器 healthy（约 1-2 分钟）
docker compose ps
```

验证：访问 http://localhost:80 可看到 RAGFlow Web UI。

### 4.5 配置 Agent

```bash
# 安装 Python 依赖
pip install -r requirements.txt

# 创建 .env 配置
cp .env.example .env

# 获取 RAGFlow API Key 和数据集 ID（自动）
python setup_apitoken.py

# 编辑 .env，填入以下内容：
# DASHSCOPE_API_KEY=你的通义千问API_Key（手动申请）
# RAGFLOW_API_KEY=ragflow-xxxx（上一步输出）
# RAGFLOW_DATASET_IDS=xxx（上一步输出）
```

### 4.6 启动 Agent

```powershell
# PowerShell
.\start_server.ps1

# 或 Git Bash
bash start.sh
```

## 八、常见问题

### Q1: setup.ps1 运行后提示需要重启

首次启用 WSL2 需要重启一次。重启后再次右键运行 `setup.ps1`，脚本会自动从断点继续。

### Q2: Docker Desktop 启动超时

- 确认 BIOS 中虚拟化（VT-x/AMD-V）已启用
- 手动打开 Docker Desktop，等待其完全启动后重新运行 `setup.ps1`

### Q3: RAGFlow 容器启动失败

- 确认 WSL2 内存配置至少 12GB（检查 `~\.wslconfig`）
- 运行 `docker compose logs ragflow` 查看日志
- 确认镜像已导入：`docker images | grep ragflow`

### Q4: 知识库为空（数据导入后无数据）

- 确认数据卷名称正确（前缀 `ragflow_docker_`）
- 数据导入必须在容器停止时进行
- 重新运行 `setup.ps1` 会自动处理

### Q5: RAGFlow 可用但知识检索无结果

- 确认 `.env` 中 `RAGFLOW_DATASET_IDS` 正确
- 确认 `RAGFLOW_API_KEY` 正确
- 在 Web UI (http://localhost:80) 确认文档解析状态正常

### Q6: Agent 回答不引用知识库

- 确认 RAGFlow 5 个容器全部运行：`cd ragflow_docker && docker compose ps`
- 测试 RAGFlow API：`curl http://127.0.0.1:9380/api/v1/datasets`
- 查看 `server.log` 中是否有 RAGFlow 调用报错

### Q7: Docker 磁盘空间不足

- RAGFlow 完整部署约占 10-15GB
- Windows 默认 Docker 数据在 C 盘，可在 Docker Desktop 设置中迁移

### Q8: WMS 接口超时

- 确认 WMS 服务器可达
- 如网络较慢可在 `.env` 中调大 `LLM_TIMEOUT`
