# 粮情分析智能体 (Grain Agent V008) — 交付部署指南

## 一、交付物清单

| 文件夹 | 内容 | 大小 |
|--------|------|------|
| `grain_agent-ubuntu/V008/` | Agent 后端源码 + 知识库原始资料 + 测试集 | ~200 MB |
| `ragflow/docker/` | RAGFlow 部署配置（docker-compose.yml 等） | ~5 MB |
| `ragflow_export/` | Docker 镜像 + 已解析知识库数据卷（免拉取、免解析） | ~4.4 GB |

### ragflow_export 包内容

| 文件 | 大小 | 说明 |

| `vol_minio.tar.gz` | ~339 MB | MinIO 数据（上传的原始文件副本） |
| `vol_redis.tar.gz` | ~16 KB | Redis 缓存数据 |
| `import.sh` | - | 一键导入脚本 |
| `README.md` | - | 导入包说明文档 |

> 注意：交付前请删除 `V008/.env` 中的真实 API Key，或提供 `.env.example` 模板。

## 二、环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11（需启用 WSL2）或 Linux |
| Python | 3.10+ |
| Docker Desktop | 29.x+（Windows）或 Docker Engine 24+（Linux） |
| 内存 | 建议 16GB+，WSL2 分配至少 12GB |
| 磁盘 | RAGFlow 容器约占 10-15GB |
| 网络 | 需访问通义千问 API（dashscope.aliyuncs.com）和 WMS 接口（121.40.162.1:8017） |

> 不需要能访问 Docker Hub（镜像已包含在交付包中）。

### WSL2 内存配置（Windows）

创建或编辑 `C:\Users\<用户名>\.wslconfig`：

```ini
[wsl2]
memory=12GB
```

保存后执行 `wsl --shutdown` 重启 WSL。

## 三、部署步骤

### 第 1 步：导入 Docker 镜像

```bash
cd ragflow_export

# 导入 5 个镜像（无需联网，约 2-5 分钟）
docker load -i images.tar
```

验证镜像已导入：
```bash
docker images | grep -E "ragflow|mysql|elasticsearch|minio|valkey"
```
应看到 5 个镜像。

### 第 2 步：启动 RAGFlow 容器

```bash
cd ragflow/docker

# 切换到与镜像匹配的版本（重要！main 分支与 v0.24.0 镜像不兼容）
git checkout v0.24.0

# 启动 5 个容器（MySQL, Elasticsearch, MinIO, Redis, RAGFlow）
docker compose up -d
```

等待 1-2 分钟，确认所有容器健康：
```bash
docker compose ps
```
应看到 5 个容器状态为 `Up` 或 `healthy`。

### 第 3 步：导入知识库数据

先停止容器，导入数据后重新启动：

```bash
# 停止容器（保留数据卷）
docker compose down

# 创建数据卷（如果不存在）
docker volume create docker_mysql_data
docker volume create docker_esdata01
docker volume create docker_minio_data
docker volume create docker_redis_data

# 导入 4 个数据卷（将 /path/to/ragflow_export 替换为实际路径）
docker run --rm -v docker_mysql_data:/data -v /path/to/ragflow_export:/backup alpine sh -c "cd /data && tar xzf /backup/vol_mysql.tar.gz"
docker run --rm -v docker_esdata01:/data -v /path/to/ragflow_export:/backup alpine sh -c "cd /data && tar xzf /backup/vol_es.tar.gz"
docker run --rm -v docker_minio_data:/data -v /path/to/ragflow_export:/backup alpine sh -c "cd /data && tar xzf /backup/vol_minio.tar.gz"
docker run --rm -v docker_redis_data:/data -v /path/to/ragflow_export:/backup alpine sh -c "cd /data && tar xzf /backup/vol_redis.tar.gz"

# 重新启动
docker compose up -d
```

> 如果 `alpine` 镜像无法拉取（网络不通），可先用交付方提供的任意本地镜像替代，
> 或参考 `ragflow_export/README.md` 中的备用导入方法。

### 第 4 步：验证知识库

1. 访问 http://localhost:80 登录 RAGFlow Web UI
2. 进入"数据集"页面，应看到"粮食"数据集（167 个文档，4001 个 chunks）
3. 随意点开一个文档，确认解析状态正常

> 登录账号使用原始注册的邮箱和密码。如需新建账号，直接在登录页注册即可。

### 第 5 步：获取 RAGFlow API Key

1. 在 RAGFlow Web UI 右上角点击头像 → 用户设置
2. 复制 API Key（格式：`ragflow-xxxx`）

同时记录数据集 ID：进入"数据集" → "粮食"，在浏览器地址栏 URL 中可看到 ID。

### 第 6 步：配置 Agent 后端

```bash
cd V008

# 安装 Python 依赖
pip install -r requirements.txt
```

编辑 `V008/.env` 文件，填入以下配置：

```env
# === 必须修改 ===
# 通义千问 API Key（申请地址: https://dashscope.console.aliyun.com/）
DASHSCOPE_API_KEY=你的通义千问API_Key

# RAGFlow API Key（第 5 步获取）
RAGFLOW_API_KEY=你的RAGFlow_API_Key

# RAGFlow 数据集 ID（第 5 步记录，多个用逗号分隔）
RAGFLOW_DATASET_IDS=你的数据集ID

# === 通常不需要修改 ===
LLM_MODEL=qwen-max
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
EMBEDDING_MODEL=text-embedding-v3
EMBEDDING_DIMENSIONS=1024
RAGFLOW_BASE_URL=http://127.0.0.1:9380
RAG_SIMILARITY_THRESHOLD=0.1
RAG_TOP_K=5
RAG_PAGE_SIZE=15
RAG_RERANK_ID=gte-rerank
RAG_ENABLE_QUERY_REWRITE=false
DEBUG=false
```

> `RAG_*` 参数为经消融实验验证的最优配置（rerank_clean），一般无需调整。
> 详见 `V008/RAG升级.md`。

### 第 7 步：启动 Agent 后端

```bash
cd V008

# 方式一：直接启动
uvicorn app.main:app --host 0.0.0.0 --port 8000

# 方式二：使用启动脚本
# Windows PowerShell:
.\start_server.ps1
# Linux/Mac:
bash start_server.sh
```

访问 http://localhost:8000 确认返回：
```json
{"message": "Grain Agent V008 is running", ...}
```

## 五、系统架构简述

```
用户 → FastAPI (port 8000)
         ├── AgentService（意图识别 + 工具调用）
         │     ├── GrainTools (T1-T8)：查询/分析/预测/图表/报告
         │     │     └── WMS Client → WMS API (121.40.162.1:8017)
         │     ├── knowledge_search → RAGFlow (port 9380)
         │     └── LLM Service → 通义千问 API
         └── 静态文件 /artifacts/（生成的图表和报告）
```

### 核心模块

| 文件 | 职责 |
|------|------|
| `app/main.py` | FastAPI 入口 |
| `app/services/agent_service.py` | Agent 主服务：意图识别 + 工具调度 |
| `app/services/tools.py` | 粮情工具集 T1-T8（查询/分析/预测/图表/报告） |
| `app/services/tool_definitions.py` | 工具 JSON Schema 定义（供 LLM Function Calling） |
| `app/services/rag_service.py` | RAGFlow 知识检索服务（三层架构 + rerank） |
| `app/services/llm_service.py` | LLM 服务（通义千问，含 Function Calling） |
| `app/services/wms_client.py` | WMS 接口客户端 |
| `app/services/predictor/` | 温度预测模块（空间预测器 + 数据验证器） |
| `app/core/config.py` | 配置管理（从 .env 加载） |
| `app/core/wms_endpoints.py` | WMS API 端点定义 |

### 关键配置文件

| 文件 | 说明 |
|------|------|
| `.env` | 环境变量（API Key、RAGFlow 配置、RAG 参数） |
| `data/knowledge/` | 知识库原始资料（167个粮储标准文档，备份用） |
| `data/eval/` | 评估测试集和消融实验结果 |
| `RAG升级.md` | RAG 检索优化报告（rerank_clean 消融实验详情） |

## 六、常见问题

### Q1: RAGFlow 容器启动失败

- 检查 Docker Desktop 是否运行
- 确认 WSL2 内存配置至少 12GB
- 确认已执行 `docker load -i images.tar` 导入镜像
- 运行 `docker compose logs ragflow` 查看日志
- 如果镜像损坏，重新执行 `docker load -i images.tar`

### Q2: 导入数据卷后知识库为空

- 确认导入时容器已停止（`docker compose down`）
- 确认数据卷名称正确（`docker_mysql_data`、`docker_esdata01`、`docker_minio_data`、`docker_redis_data`）
- 导入后需执行 `docker compose up -d` 重启

### Q3: alpine 镜像拉不下来（数据卷导入失败）

数据卷导入命令使用了 `alpine` 镜像，如果无法从 Docker Hub 拉取：
- 可用任何已有的带 `tar` 命令的镜像替代（如 `ubuntu`、`debian`）
- 或用已运行的容器手动导入（参考 `ragflow_export/README.md`）

### Q4: RAGFlow 可用但知识检索无结果

- 确认 `.env` 中 `RAGFLOW_DATASET_IDS` 与 Web UI 中数据集 ID 一致
- 确认 `RAGFLOW_API_KEY` 正确
- 在 Web UI 中确认文档解析状态正常

### Q5: Agent 回答不引用知识库

- 确认 RAGFlow 5 个容器全部运行：`docker compose ps`
- 测试 RAGFlow API 是否可达：`curl http://127.0.0.1:9380/api/v1/datasets`
- 查看 Agent 日志中是否有 RAGFlow 调用报错

### Q6: PDF 文档解析乱码

- 部分 PDF（如扫描件）需要 OCR 解析模式
- 在 RAGFlow Web UI 中对该文档切换解析方法为 OCR 模式重新解析
- 或在"模型供应商"中配置 OCR 模型

### Q7: Docker 磁盘空间不足

- Windows 默认 Docker 数据在 C 盘，可迁移到其他盘
- RAGFlow 完整部署约占 10-15GB 磁盘空间
- 迁移后如遇镜像损坏，重新执行 `docker load -i images.tar`

### Q8: WMS 接口超时

- 当前超时设置为 5 秒（`app/core/wms_endpoints.py` 中 `WMS_API_TIMEOUT`）
- 如网络较慢可适当调大
- 确认 WMS 服务器可达：`curl http://121.40.162.1:8017`

## 七、启动/停止速查

```bash
# ── 启动（按顺序）──
# 1. 启动 RAGFlow（进入 ragflow/docker 目录）
cd ragflow/docker && docker compose up -d

# 2. 启动 Agent 后端
cd V008 && uvicorn app.main:app --host 0.0.0.0 --port 8000

# ── 停止（按顺序）──
# 1. 停止 Agent 后端
Ctrl+C

# 2. 停止 RAGFlow
cd ragflow/docker && docker compose down
```
