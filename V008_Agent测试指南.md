# V008 Agent 测试指南

> 所有问答测试都通过 V008 Agent 进行，不需要打开 RAGFlow 页面。
> 测试脚本：`V008/test_requests_qa.py`

---

## 前提条件

| 条件 | 说明 |
|------|------|
| Docker Desktop | 必须启动，且 5 个 RAGFlow 容器处于 running 状态（知识问答需要） |
| V008 Agent 服务 | 必须在端口 8000 运行 |

---

## 第一步：启动 Docker（如果未启动）

1. 双击桌面 **Docker Desktop**，等待左下角变绿
2. 打开终端，启动 RAGFlow 容器：

```bash
cd C:\Users\ASUS\Desktop\ragflow\docker
docker compose up -d
```

3. 验证容器状态：

```bash
docker ps
```

确认 5 个容器（ragflow、mysql、es01、minio、redis）都是 running。

---

## 第二步：启动 V008 Agent 服务

打开一个**新的终端窗口**（不要关闭 Docker 的终端），执行：

```bash
cd C:\Users\ASUS\Desktop\grain_agent-ubuntu\V008
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

看到类似以下输出表示启动成功：

```
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

> **注意**：这个终端窗口会被 Agent 服务占用，不要关闭它。

---

## 第三步：运行测试脚本

再打开一个**新的终端窗口**，执行：

```bash
cd C:\Users\ASUS\Desktop\grain_agent-ubuntu\V008
python test_requests_qa.py
```

---

## 第四步：使用测试脚本

### 启动后界面

```
============================================================
V008 版本交互式 API 测试
============================================================
[OK] 服务器运行中

[INFO] 可用仓号: 15, 18, ...
[INFO] 数据时间范围: 2026-01-01 到 2026-01-05
```

### 选择仓号

如果有多个仓号，脚本会列出所有可用仓号，输入序号选择。

### 选择查询类型

```
请选择查询类型:
  1. 日报查询        ← 生成指定时间段的粮温日报
  2. 趋势分析        ← 分析粮温变化趋势
  3. 图表生成        ← 生成三温图和两湿图
  4. 巡检查询        ← 巡检粮温情况
  5. 自定义查询      ← 输入任意问题（知识问答用这个）
  0. 退出
```

### 测试粮温数据查询（选 1-4）

选择 1-4 后，输入时间范围：

```
时间范围: 2026年1月
```

脚本会自动构建查询语句并发送给 Agent。

### 测试知识问答（选 5）

选择 5 后，直接输入你的问题：

```
请输入查询语句: 粮食安全储存的温度标准是什么？
```

Agent 会自动识别为知识问答意图，调用 RAGFlow 检索知识库并回答。

```

------------------------------------------------------------
```

---

## 退出测试

- 在"请选择查询类型"时输入 `0`
- 或在"是否继续查询"时输入 `n`

---

## 停止服务

测试完毕后：

1. 在运行 Agent 服务的终端按 `Ctrl + C` 停止 Agent
2. 关闭 RAGFlow 容器（可选）：

```bash
cd C:\Users\ASUS\Desktop\ragflow\docker
docker compose down
```

---

## 常见问题

| 问题 | 解决方案 |
|------|---------|
| 脚本提示"服务器未运行" | 先执行第二步启动 V008 Agent |
| 知识问答回答"未找到相关信息" | 检查 Docker 和 RAGFlow 容器是否运行 |
| 请求超时（>120秒） | PDF 解析或 LLM 响应较慢，检查网络和系统资源 |
| 中文乱码 | Windows 终端编码问题，尝试在终端执行 `chcp 65001` 切换到 UTF-8 |

---

## 总结：需要开几个终端窗口

| 终端 | 用途 | 保持运行 |
|------|------|---------|
| 终端 1 | Docker 容器（RAGFlow） | 是（不要关闭 Docker Desktop） |
| 终端 2 | V008 Agent 服务 | 是（`Ctrl+C` 停止） |
| 终端 3 | 运行测试脚本 | 测试时使用 |
