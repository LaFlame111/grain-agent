# test_requests_qa.py 使用说明

## 概述

`test_requests_qa.py` 是一个交互式 API 测试脚本，允许用户通过键盘输入时间区间进行粮情数据查询和分析。

## 功能特点

1. **交互式时间输入**: 支持多种时间格式输入
2. **多种查询类型**: 日报、趋势分析、图表生成、巡检等
3. **自动时间解析**: 自动将用户输入转换为查询时间范围
4. **真实数据支持**: 使用 WMS 真实数据（示例时间范围：2026-01-01 到 2026-01-05）

## 使用方法

### 1. 启动服务器

首先确保 V008 服务器正在运行：

```powershell
cd V008
.\start_server.ps1
```

### 2. 运行测试脚本

```powershell
cd V008
python test_requests_qa.py
```

### 3. 交互式操作

脚本启动后会显示：

```
============================================================

============================================================
[OK] 服务器运行中

[INFO] 可用仓号: 15
[INFO] 数据时间范围: 2026-01-01 到 2026-01-05

============================================================
查询类型说明:
============================================================
1. 日报查询 (daily) - 生成指定时间段的日报

3. 图表生成 (chart) - 生成三温图和两湿图
4. 巡检查询 (inspection) - 巡检指定时间段的粮温情况
```

## 支持的时间格式

### 单月查询

输入格式：`2026年1月`

示例：
```
时间范围: 2026年1月
```

解析结果：2026-01-01 00:00:00 到 2026-01-31 23:59:59

### 月份范围查询

输入格式：`2026年1月-2026年12月`

示例：
```
时间范围: 2026年1月-2026年12月
```

解析结果：2026-01-01 00:00:00 到 2026-12-31 23:59:59

### 年度查询

输入格式：`2026年`

示例：
```
时间范围: 2026年
```

解析结果：2026-01-01 00:00:00 到 2026-12-31 23:59:59

### 跳过时间范围

直接按回车键，将查询所有可用数据。

## 查询类型说明

### 1. 日报查询 (daily)

生成指定时间段的粮情分析日报，包含：
- 粮温统计信息
- 三温图和两湿图
- 分析结论和建议

**示例查询**：
- 时间范围：`2026年1月`
- 生成的查询：`生成P1号仓在2026年01月的日报`

### 2. 趋势分析 (trend)

分析指定时间段的粮温变化趋势，包含：
- 温度变化趋势图
- 趋势分析结论

**示例查询**：
- 时间范围：`2026年1月-2026年12月`
- 生成的查询：`分析P1号仓从2026年01月01日到2026年12月31日的粮温变化趋势`

### 3. 图表生成 (chart)

生成三温图和两湿图，包含：
- 三温图（气温、仓温、粮温）
- 两湿图（气湿、仓湿）

**示例查询**：
- 时间范围：`2026年1月`
- 生成的查询：`给我生成P1号仓在2026年01月的三温图和两湿图`

### 4. 巡检查询 (inspection)

巡检指定时间段的粮温情况，包含：
- 异常检测
- 风险提示

**示例查询**：
- 时间范围：`2026年1月`
- 生成的查询：`巡检P1号仓在2026年01月的粮温情况`

### 5. 自定义查询 (custom)

直接输入自然语言查询语句，系统会自动识别意图并调用相应工具。

**示例查询**：
- `P1号仓在2026年1月的平均粮温是多少？`
- `查询P1号仓在2026年的最高温度和最低温度`
- `给我分析一下P1号仓在2026年1月的粮温异常情况`

## 注意事项

1. **时间范围**: 确保输入的时间范围在真实数据范围内（示例：2026-01-01 到 2026-01-05）
2. **时间格式**: 严格按照格式输入，例如 `2026年1月` 而不是 `2026-01` 或 `2026/1`
3. **服务器状态**: 确保服务器正在运行，否则会提示连接错误
4. **查询超时**: 报告生成可能需要较长时间（最长120秒），请耐心等待

## 故障排除

### 问题1: 服务器未运行

**错误信息**:
```
❌ 错误: 服务器未运行！
```

**解决方法**:
```powershell
cd V008
.\start_server.ps1
```

### 问题2: 时间格式解析失败

**错误信息**:
```
❌ 时间格式解析失败，请检查格式
```

**解决方法**: 检查输入格式是否正确，例如：
- ✅ 正确: `2026年1月`
- ❌ 错误: `2026-01`、`2026/1`、`2026年1`

### 问题3: 查询超时

**错误信息**:
```
❌ 超时错误: 请求超时（>120秒）
```

**解决方法**: 
- 检查服务器日志
- 尝试缩小时间范围
- 检查网络连接

## 技术细节

### 时间解析逻辑

脚本使用正则表达式解析时间字符串：

1. **单月格式**: `r'(\d{4})年(\d{1,2})月'`
2. **月份范围**: `r'(\d{4})年(\d{1,2})月-(\d{4})年(\d{1,2})月'`
3. **年度格式**: `r'(\d{4})年$'`

### 查询构建

根据查询类型和时间范围，自动构建自然语言查询语句，然后发送给 Agent API。

### 数据来源

脚本会自动检测并使用真实数据（`data/grain_data_wms_format.json`），如果数据加载失败，会提示使用 Mock 数据模式。
