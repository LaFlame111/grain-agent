# test_requests.py 使用示例

## 概述

`test_requests.py` 是 V008 版本的 API 测试脚本，用于测试 Agent API 接口，使用真实 WMS 数据。

## 前置条件

1. **Python 环境**：确保已安装 Python 3.8+ 和所需依赖
2. **数据文件**：确保 `data/grain_data_wms_format.json` 文件存在
3. **环境变量**：确保已设置 `DASHSCOPE_API_KEY`（用于 LLM 服务）

## 使用步骤

### 步骤 1: 启动服务器

在第一个终端窗口中，启动 V008 服务器：

```powershell
# Windows PowerShell
cd V008
.\start_server.ps1 -HostAddress 0.0.0.0 -Port 8000
```

```bash
cd V008
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

## 测试场景说明

脚本会自动执行以下4个测试场景：

1. **单仓查询**：查询指定仓号在特定月份的粮温情况
2. **时间范围查询**：查询指定仓号在多年时间范围内的数据
3. **三温图生成**：生成指定仓号的三温图（气温、仓温、粮温）
4. **日报生成**：生成指定仓号的日报报告

## 常见问题

### 问题 1: 服务器未运行

**错误信息**：
```
❌ 错误: 服务器未运行！
请先启动服务器:
  cd V008
  .\start_server.ps1
```

**解决方案**：
- 确保服务器已启动（参考步骤 1）
- 检查服务器是否在端口 8000 上运行：`netstat -an | findstr 8000`

### 问题 2: 无法加载数据

**错误信息**：
```
[WARN] 无法导入 DataLoader，将使用默认仓号
[WARN] 无法获取数据时间范围，查询可能会使用Mock数据
```

**解决方案**：
- 确保 `data/grain_data_wms_format.json` 文件存在
- 检查文件路径是否正确（相对于项目根目录）
- 确保数据文件格式正确（JSON 格式）

### 问题 3: 连接超时

**错误信息**：
```
❌ 超时错误: 请求超时（>60秒）
```

**解决方案**：
- 检查服务器日志，查看是否有错误
- 确保 LLM API 密钥已正确设置
- 检查网络连接是否正常

### 问题 4: 查询返回 Mock 数据

**现象**：查询结果看起来像是模拟数据，而不是真实的历史数据

**原因**：
- 查询的时间范围不在真实数据范围内（示例：2026-01-01 到 2026-01-05）
- Agent 服务无法识别查询中的时间范围

**解决方案**：
- 确保查询语句中明确指定了历史数据的时间范围
- 检查脚本是否正确提取了数据时间范围
- 查看服务器日志，确认数据加载情况

## 注意事项

1. **数据时间范围**：V008 版本使用真实 WMS 数据，推荐使用 `2026-01-01` 到 `2026-01-05` 进行验证
2. **仓号格式**：建议使用真实 WMS 列表中的短名（如 "P1"、"Q1"）
3. **服务器状态**：确保服务器在测试期间保持运行
4. **API 密钥**：确保 `DASHSCOPE_API_KEY` 环境变量已正确设置
